# 大型資料檔案編輯工具

這是使用 C#、.NET 8 和 WPF 建立的 Windows 大型分隔文字檔工具。目前已完成四個階段：檔案選擇、背景掃描、CSV/TSV/分號/自訂單一字元分隔符號、欄位標題清單、進度顯示、取消掃描、串流刪除單一資料列、依標題或位置串流刪除單一欄位，以及安全取代原始檔與備份流程。

## 程式用途

此工具用於處理數 GB 以上的 CSV、TSV 或其他分隔符號文字檔。設計原則是不在視窗中顯示完整檔案內容，也不將完整檔案載入 RAM，只顯示檔案資訊與欄位資訊。

## 支援的檔案格式

- CSV，預設逗號分隔
- TSV，Tab 分隔
- 分號分隔文字檔
- 使用者指定的單一字元分隔符號

## CSV 解析規則

第一階段內建串流 CSV parser，不使用 `string.Split(',')`。目前支援：

- 雙引號包住的欄位
- 雙引號欄位內的分隔符號
- 雙引號欄位內的換行
- 兩個連續雙引號代表欄位內的一個雙引號
- 空白欄位
- 最後一欄為空值
- 最後一列沒有結尾換行
- 不同資料列欄位數不一致

## 資料列編號計算方式

第一列是欄位標題時，掃描結果同時顯示「含標題列」與「不含標題列」資料列數。刪除資料列功能以不含標題列的資料內容列作為預設列號基準，列號從 1 開始。

## 如何刪除資料列

在下方操作區輸入要刪除的資料列編號與輸出檔案路徑，按「刪除資料列」。第二階段支援刪除單一資料列並另存新檔。處理過程使用 `long` 列號、串流讀取原始檔並輸出到新檔案，不會直接修改原始檔，也不會將所有資料列載入記憶體。取消時會刪除未完成輸出檔，原始檔保持不變。

## 如何刪除欄位

在欄位清單選取欄位，或在下方操作區輸入欄位標題或欄位位置，按「刪除欄位」。欄位位置從第 1 欄開始，內部以 0-based index 處理。標題比對預設完全相符，可選擇忽略英文大小寫或忽略標題前後空白。

若存在多個同名欄位，程式不會自動刪除全部同名欄位，必須指定實際欄位位置。資料列欄位數少於刪除位置時，該列維持原欄位；資料列欄位數多於標題列時，額外欄位會保留，除非被刪除的位置正好落在該額外欄位。

刪除欄位後會以 CSV 規則重新輸出，包含分隔符號、雙引號或換行的欄位會加上引號並跳脫。

## 原始檔保護機制

掃描只讀取檔案，不修改原始檔。刪除資料列或欄位預設另存新檔。若勾選「完成後取代原始檔」，程式會：

1. 在原始檔同目錄建立暫存輸出檔。
2. 串流寫入完成並重新開啟驗證。
3. 將原始檔改名為 `filename.backup.csv` 形式的備份檔。
4. 將暫存檔移成原始檔名。
5. 詢問是否保留備份檔。
6. 若替換失敗，嘗試將備份檔移回原始檔名。

作業開始前會檢查目標磁碟可用空間，空間不足時不會開始重寫。

## 建置與執行方式

```powershell
dotnet build LargeFileEditor.App\LargeFileEditor.App.csproj -c Release
dotnet run --project LargeFileEditor.App\LargeFileEditor.App.csproj
```

## 測試

此工作區使用不依賴 NuGet 套件的 console 測試執行器：

```powershell
dotnet run --project LargeFileEditor.Tests\LargeFileEditor.Tests.csproj -c Release
```

測試涵蓋 UTF-8 無 BOM、UTF-8 BOM、UTF-16 LE、UTF-16 BE、CRLF、LF、空檔案、只有標題列、沒有標題列、空白欄位、重複欄位標題、引號欄位、欄位內換行、最後欄位空值、最後列無結尾換行、不一致欄位數、取消掃描、大型測試檔產生器、刪除第一筆/中間/最後資料列、不存在資料列、取消刪除時原始檔保持完整、依標題/位置刪除欄位、刪除第一欄/最後一欄、同名欄位處理、CSV 重新輸出跳脫、取消欄位刪除，以及安全替換原始檔與備份處理。

## 大型檔案測試方法

`LargeTestFileGenerator` 使用 `FileStream` 和 `StreamWriter` 串流產生 CSV，可指定資料列數、欄位數與目標近似大小，不會先把全部內容放進記憶體。

## 已知限制

- 目前刪除資料列僅支援單一資料列，尚未支援 `1-100,500` 這類範圍語法。
- 刪除欄位目前支援單一欄位。
- 安全取代原始檔目前使用檔案改名與移動流程；若原始檔被其他程式鎖定，作業會失敗並回報原始檔安全狀態。
- 目前自動編碼偵測以 BOM 為主；無 BOM 檔案預設以 UTF-8 讀取。
- 第一階段只偵測欄位數最大值與標題資訊，尚未在 UI 顯示每一列欄位數不一致的詳細位置。
